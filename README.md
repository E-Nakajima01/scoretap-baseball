# ScoreTap Baseball

初心者向けの野球スコアブックアプリです。`K`、`BB`、`F8`、`6-3` などの伝統的な記号を知らなくても、大きなボタンと守備位置タップでプレーを記録できます。

## 想定する使い方

- 少年野球、草野球、部活動、地域チームなどで、チーム内の試合記録を継続して残す
- 保護者、マネージャー、コーチなど、伝統的なスコアブックに慣れていない人が記録する
- 1試合の結果だけでなく、あとからチームや選手の振り返りに使う
- 相手チームの詳細メンバーがわからない試合でも、最低限の試合記録として残す

## 主な機能

- ホームチーム、アウェイチーム、日付、打順を登録
- ホーム画面から新しい試合記録を作成
- 保存済みの試合結果一覧を表示
- チーム名とメンバーを登録して、試合作成時に呼び出し
- メールなしで初期ログインIDと初期パスワードを発行し、IDとパスワードでログイン
- ログインIDはあとから変更可能
- チームごとに招待コードを発行し、コード入力でチーム参加
- チーム詳細で試合数、勝敗、選手一覧を表示
- 選手詳細で打率、出塁率、長打率、OPS、安打、本塁打、四死球、三振、盗塁などを表示
- 投手登板がある選手には、投手データとして投球回、防御率、自責点、WHIP、K/BB、被安打、四死球、奪三振などを表示
- 選手詳細の関連プレーから、打席ごとの1球速報風の内容を表示
- 打席数と打数を分け、P/PA、BB%、K%、BB/K、F/PAなどを選手指標として表示
- 予定イニング、DH制、コールド条件を試合前に設定
- DH制のときは打順とは別に投手を登録
- イニング、スコア、アウト、ボール/ストライク、走者、現在の打者を表示
- BSOランプとダイヤモンド型の塁表示で試合状況を視覚的に表示
- ボール、見逃しストライク、空振りストライク、ファウルを1球ごとに記録
- 4ボールで四球、3ストライクで三振を自動記録
- シングル、二塁打、三塁打、ホームラン、四球、デッドボール、三振、ゴロアウト、フライアウト、エラー、犠牲バント、犠牲フライ、ダブルプレーを記録
- 守備位置をフィールド図からタップして選択
- 守備位置ごとの選手名を登録
- 試合中の守備交代を履歴に保存
- 現在の打者を代打・交代選手に変更して履歴に保存
- 9回以降に決着している試合を自動終了
- 手動終了、コールド・途中終了に対応
- 塁上のランナーをタップして盗塁、進塁、ホームイン、暴投、捕逸、盗塁死、走塁アウトを記録
- 得点を手動で微調整
- 誤って押した試合終了を取り消し
- 試合記録を一覧から削除
- チーム登録を編集・削除
- 日本語の説明と伝統的なスコアコードを同時に保存
- アウト、走者、得点、次打者を自動更新
- プレー履歴を表示
- `localStorage` に自動保存

## セットアップ

```bash
npm install
npm run dev
```

ブラウザで `http://localhost:3000` を開いてください。

## リリース前チェック

```bash
npm run verify
```

`verify` は型チェック、Lint、本番ビルドをまとめて実行します。

## スマホで使う

- スマホではブラウザで公開URLを開きます。
- iPhoneはSafariの共有メニューから「ホーム画面に追加」を選ぶと、アプリのように起動できます。
- AndroidはChromeのメニューから「ホーム画面に追加」または「アプリをインストール」を選びます。
- PWA用のマニフェスト、アイコン、Service Workerを入れているため、2回目以降は読み込みが軽くなります。

## 公開方法

まずは Vercel にデプロイするのが簡単です。

1. GitHubにこのプロジェクトを置きます。
2. Vercelでリポジトリを選択します。
3. Build Commandは `npm run build`、OutputはNext.js標準のままで公開します。
4. 公開URLをスマホで開き、ホーム画面に追加します。

現時点のユーザー、チーム、試合データは `localStorage` に保存されます。スマホを変えたりブラウザのデータを消したりすると消えるため、実運用でチーム共有する前にSupabaseなどのクラウドDBへ移行するのが次の大きなステップです。

## Supabaseでクラウド保存する

1. Supabaseで新しいProjectを作成します。
2. Project Settings → API で以下をコピーします。
   - Project URL
   - anon public key
3. SQL Editorで [supabase/schema.sql](./supabase/schema.sql) のSQLを実行します。
4. Authentication → Providers → Email で、メール確認をOFFにします。
   ScoreTapは画面上メールを使わず、内部的に `loginId@scoretap.local` の合成メールでSupabase Authを使います。
5. ローカルでは `.env.local` を作り、以下を設定します。

```bash
NEXT_PUBLIC_SUPABASE_URL=あなたのProject URL
NEXT_PUBLIC_SUPABASE_ANON_KEY=あなたのanon public key
```

6. Vercelでも Project Settings → Environment Variables に同じ2つを追加します。
7. Vercelで再デプロイします。

Supabase環境変数がない場合は、今まで通り `localStorage` 保存で動きます。環境変数がある場合は、ログイン後にチーム、試合一覧、現在の試合を `scoretap_cloud_state` に同期します。

## 使い方

1. ホーム画面の「ログイン」で、発行済みのIDとパスワードを入力します。
   はじめて使う場合は「IDと初期パスワードを発行」を押します。
2. ログイン後、必要ならアカウント設定でID、表示名、パスワードを変更します。
3. 既存チームに参加する場合は、チーム招待コードを入力します。
4. ホーム画面で「新しい試合記録を作る」を押します。
5. よく使うチームはホーム画面の「チーム・メンバー登録」で先に保存できます。
   メンバーは名前を入力して「追加」を押すと登録リストに入ります。
6. 試合設定画面でチーム名、日付、予定イニング、DH制、コールド条件、スタメンを入力します。
   スタメンは打順の横で守備位置を選べます。
   DH制をオンにした場合は、打順とは別に投手名も入力します。
7. 「スコア入力を開始」を押します。
8. 1球ごとに「ボール」「見逃し」「空振り」「ファウル」を押します。
9. ゴロ、フライ、エラーなど守備位置が関係するプレーでは、先にフィールド図の守備位置をタップします。
10. 打席結果の大きなボタンを押すと、説明とコードが履歴に追加されます。
11. 守備交代があったら、守備位置を選んで交代後の選手名を入力します。
12. 代打など攻撃側の交代があったら、現在の打者の交代後の選手名を入力します。
13. 盗塁や走塁は、塁上のランナーをタップして「盗塁」「進塁」「ホームへ」「盗塁死」「アウト」を押します。
14. 得点が実際のプレーと違った場合はランナー操作内の得点補正で直します。
15. 9回以降に決着している場合は自動で試合終了になります。途中で終わる場合は「試合終了」または「コールド終了」を押します。
16. 誤って試合終了した場合は「試合終了を取り消す」で記録中に戻せます。

## アカウント方針

- 内部的には変更されない `internalUserId` でデータを紐付けます。
- ユーザーがログインに使う `loginId` は変更可能です。
- 共有はユーザーIDではなく、チーム招待コードで行います。
- 現在は `localStorage` の試作実装です。本番ではパスワードをサーバー側で安全にハッシュ化し、復旧コードや管理者による再発行で復旧できるようにします。

## 取り消しと削除

- 試合終了はスコア画面から取り消せます。
- 試合結果一覧の「削除」で試合記録を削除できます。
- チーム・メンバー登録の `⋯` から編集・削除できます。既存の試合記録は残ります。
- 「直近履歴だけ削除」は履歴表示から1件消す簡易操作です。スコア、走者、アウトまで完全に戻すには今後スナップショット式のUndoが必要です。

## 成績指標について

- 打者は、打率だけでなく出塁率、長打率、OPSを重視して表示します。OPSは出塁率と長打率を足した総合的な打力の目安です。
- 選球眼や粘りを見るために、P/PA、BB%、K%、BB/K、F/PAを表示します。
- 投手は、投球回、防御率、自責点、RA/9、WHIP、K/BB、K%、BB%、BF、P/BFを表示します。
- WHIPは1投球回あたりに出した安打と四球、RA/9は全失点を9回あたりに直した数字です。
- 防御率は `自責点 x 9 ÷ 投球回` で計算します。
- エラーで出塁した走者、エラーで進塁した走者、捕逸で進塁・得点した走者、エラーがなければ3アウトになっていた後の得点は非自責として扱います。
- 暴投、四球、死球、安打、犠打、犠飛、盗塁による得点は投手責任として自責点の対象にします。
- 既存の保存済み試合で自責点情報が残っていないものは、過去データ互換のため失点を自責点として扱います。
